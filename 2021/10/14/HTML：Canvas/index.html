HTML：Canvas

发表于 2021-10-14 更新于 2021-10-16 分类于 前端
本文字数： 5.8k 阅读时长 ≈ 15 分钟

该画画了！

HTML里要绘制图形，得用<canvas> 。它是H5新增的，用来绘制和展示图像的标签，得和 JS DOM 配合使用。除了用来画画，他也可以直接放一些图片（img）进去，并对图片做一些操作，甚至还可以配合循环计时器做出一些简单的动画。

这是简单的说明：Canvas 是由 HTML 代码配合高度和宽度属性而定义出的可绘制区域。JavaScript 代码可以访问该区域，类似于其他通用的二维 API，通过一套完整的绘图函数来动态生成图形。

基本使用

绘图，最基础的无非就是直线和曲线，画线（直线），只需要两个点。在canvas里，我们为画布设立一个坐标轴就能确定点的位置。之后就能通过各种方式画我们的线，构出我们的图。

铺一张画布：创建 canvas 元素

在HTML里创建它，它只有width，height属性，默认是 300(px) x 150(px)。虽然可以用CSS来设置宽高，但是如果宽高属性和初始比例不一致，则会变形，所以不要（也没有意义）用CSS来设置。

拿到绘画的工具箱：获取渲染上下文（getContext）

前面说了，canvas得和 JS 配合使用。只有获取了画笔，才能在画布上绘画，DOM 获取到的canvas对象会暴露出渲染上下文，相当于绘画用的画笔，我们通过这个玩意来操作canvas对象，一般都是2D的，不搞3D的

这个方法是 getContext(“2d”)

绘制图形

画一个简单的图形：矩形（Rect）

如果我们把 canvas 画布想象成一个个格子组成，每一个格子的长宽都是1px，为它设定一个坐标轴，原点是画布的左上角。这样我们就能轻松地使用坐标来表示各个点的位置。实际上，我们画图都是基于这个坐标轴来的，坐标轴也是可以移动、旋转甚至缩放的，它存在的意义就是为了我们能够准确地把图形放置在我们想要的位置上。渲染上下文提供的各种方法也是通过坐标轴来实现定位的。

我们先画一个简单的图形：矩形。事实上，canvas原生方法只有画矩形（一步到位的那种），任何其它的图形都需要自己一根线一根线画。

绘制矩形的三个方法：

  1. **fillRect(x, y, width, height)**：绘制一个填充了颜色的矩形，默认是黑色
  2. **strokeRect(x, y, width, height)**：绘制一个矩形的边框
  3. **clearRect(x, y, widh, height)**：清除指定的矩形区域，然后这块区域会变的完全透明

参数就很明显了，x 和 y 分别对应矩形左上角的坐标，width 和 height 分别对应矩形的宽和高（长和宽）

画一些稍微复杂点的图形：路径（path）

路径的定义是这样的：路径是通过不同颜色和宽度的线段或曲线相连形成的不同形状的点的集合

说人话就是，描出一个轮廓，设定好哪条线用什么颜色，画多粗，描出来的图形要不要填充颜色。这里要注意的是，路径是闭合的，无论画什么，就算是一根线，路径也总会从起点出发，最后又回到起点。

路径绘制图形的步骤如下：

  1. 创建路径起始点
  2. 调用绘制方法去绘制出路径
  3. 把路径封闭
  4. 一旦路径生成，通过描边或填充路径区域来渲染图形。

我们要用到的方法如下：

  1. beginPath()

    新建一条路径，路径一旦创建成功，图形绘制命令被指向到路径上生成路径
    如果上一条路径没结束，再次调用则会直接把上一次的路径闭合并开始一个新路径

  2. moveTo(x, y)

    把画笔移动到指定的坐标(x, y)。相当于设置路径的起始点坐标。

  3. closePath()

    闭合路径，会从当前点到起始点连一条直线路径，保证路径是闭合的。
    闭合路径之后，图形绘制命令又重新指向到上下文中

  4. stroke()

    通过线条来绘制图形轮廓，这个方法不会自动闭合路径。你可以先绘制，再闭合路径，这样就不会把闭合路径产生的那条直线路径也画出来

  5. fill()

    通过填充路径的内容区域生成实心的图形，这个方法会自动闭合路径

画直线：lineTo()

方法为**lineTo(x, y)**，从当前位置指定直线路径到指定坐标(x, y)，注意，只是确定了路径，还没有实际绘制出来

其实这里的线来回画了两次，应该不难理解。你可以试试将 ctx.closePath() 去掉，看看有什么不同。如果还不能理解，你看看下面这个：

填充的三角形是同理，注意 fill() 会自动闭合路径

画曲线：arc() ，arcTo()，quadraticCurveTo()

画曲线，最先想到的是圆弧，也就是前两个方法。如果要画更复杂的曲线，就要使用后一种贝塞尔曲线

arc(x, y, r, startAngle, endAngle, anticlockwise)

以 (x, y) 为圆心，以 r 为半径，从 startAngle 弧度开始到 endAngle 弧度结束。anticlosewise 是布尔值， true 表示逆时针，false 表示顺时针（默认是顺时针），注意这里是弧度不是度数，0弧度指x轴正方向

arcTo(x1, y1, x2, y2, radius)

根据起始点和给定的两个控制点以及半径画一段圆弧，最后再以直线连接两个控制点

这个方法可以这样理解。绘制的弧形是由两条切线所决定。

第 1 条切线：起始点和控制点1决定的直线。

第 2 条切线：控制点1 和控制点2决定的直线。

其实绘制的圆弧就是与这两条直线相切的圆弧

quadraticCurveTo(cp1x, cp1y, x, y)、bezierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y)

贝塞尔曲线，一般就找个网站先模拟出想要的图形，再直接套用参数就行了

二次贝塞尔曲线，quadraticCurveTo(cp1x, cp1y, x, y)，参数是控制点的坐标和终点的坐标

三次贝塞尔曲线，bezierCurveTo(cp1x, cp1y, cp2x, cp2y, x, y)，参数是两个控制点的坐标和终点的坐标

不多说了，我也怎么没用过，反正很强大就对了。

设置样式和颜色

全场透明：globalAlpha

globalAlpha = transparencyValue: 这个属性影响到 canvas 里所有图形的透明度，有效的值范围是 0.0 （完全透明）到 1.0（完全不透明），默认是 1.0。

在需要绘制大量拥有相同透明度的图形时候相当高效。不过，使用rgba()设置透明度更加好一些

换个颜色：fillStyle、strokeStyle

fillStyle = color 设置图形的填充颜色

strokeStyle = color 设置图形轮廓的颜色

color 可以是表示css 颜色值的字符串、渐变对象或者图案对象

一旦设置了 strokeStyle 或者 fillStyle 的值，那么这个新值就会成为新绘制的图形的默认值。如果要给每个图形上不同的颜色，需要重新设置

线条变样：lineWidth、lineCap、lineJoin、setLineDash()和lineDashOffset

lineWidth，线宽，只能是正数，单位px，由线的轴先向两边延伸

lineCap，线末端的样式，有三种样式

lineJoin，线条与线条间接合处的样式，三种样式

setLineDash()和lineDashOffset，设置虚线样式，setLineDash() 方法接受一个数组，来指定线段与间隙的交替的长度；lineDashOffset 属性设置起始偏移量

绘制文本和图片

画布当然能写字：fillText()、strokeText()

**fillText(text, x, y, maxWidth)**，在指定位置放指定文本，设定最大宽度（可选的参数）

strokeText(text, x, y, maxWidth)，在指定位置放指定文本，只不过是给文字描边，设定最大宽度（可选的参数）

添加样式：font（字体）、textAlign（文本对齐方式）、textBaseline（基线对齐方式）、direction（文本方向）

基本上就可以理解为在某位置创建一个文本框，放进文字，所以才会有一些对齐方式，坐标和宽度的参数就是设置文本框用的

当然也能放图片：drawImage()

canvas 可以放 JS 直接创建的 Image 元素（new Image()），也可以从页面里的标签取 img 元素（<img>），基本都是一样的

drawImage(image, x, y, width, height)：image为图片元素，x 和 y 是绘制的坐标，width 和 height 是可选的参数，控制图片绘制时的大小，没入参就是图片原本的大小

drawImage(image, sx, sy, sWidth, sHeight, dx, dy, dWidth, dHeight)：还可以再多参数，这可以将图片裁剪后画在canvas画布上。第一个参数是相同的，sx 和 sy 是图片源要裁切的起点，sWidth 和 sHeight 是裁切的大小，dx 和 dy 是裁剪后的图片绘制位置，dWidth 和 dHeight 是裁剪后的图片的大小。注意原本的完整的图片就不会绘制了。

保存状态和恢复

save() 和 restore() 两个方法，用来保存和读取状态。Canvas 的状态就是当前画面应用的所有样式和变形的一个快照，它存放在一个栈里，每次读取和保存就是出栈（pop）和进栈（push）

状态包括这些东西：

  • 当前应用的变形（即移动，旋转和缩放）
  • strokeStyle, fillStyle, globalAlpha, lineWidth, lineCap, lineJoin, miterLimit, shadowOffsetX, shadowOffsetY, shadowBlur, shadowColor, globalCompositeOperation
  • 当前的裁切路径（clipping path）

当我们要画很多复杂图形的时候，这个东西就显得十分有用，我们就不必反复设置各种样式和坐标轴

变形（改变坐标轴）

translate(x, y)：改变坐标轴原点的位置

rotate(angle)：旋转坐标轴

坐标轴绕原点顺时针方向旋转，参数单位是弧度，度数记得要转换

scale(x, y)：缩放坐标轴

参数都是正数，比 1 小则缩小，比 1 大则放大，是放大 / 缩小的倍数

transform(a, b, c, d, e, f)： 变形矩阵

参数分别是：scaleX，skewH，skewV，scaleY，moveX，moveY，注意顺序

这玩意有点复杂，事实上，前面所有的变形方式都可以用它来实现

scaleX 和 scaleY 同理是 1 的倍数做缩放，moveX 和 moveY 是原点离画布左上角的距离

我们要注意的是 skewX 和 skewY，可以简单地理解成 skewH 将 X 轴绕着原点向 Y 轴旋转，图形也会跟着变形，同理 skewV 将 Y 轴绕着原点向 X 轴旋转。倾斜的角度是传入的参数的**反正切值 atan(x)**，即如果传入参数 1，那么 atan(1) = 45度 = Math.PI/4 = 0.785398，即要转 45 度

具体看这里：Canvas 斜拉 - Canvas 基础教程

实现出来的效果就可以是这样，一个矩形被斜拉变形

**setTransform()**：将当前转换重置为单位矩阵（恢复初始坐标系），然后运行 transform()

合成（控制绘制图形的层级）

globalCompositeOperation，有以下取值：

  • source-over：默认，新图像会覆盖在原有图像
  • source-in：仅出现新图像与原来图像重叠的部分，其他区域都变成透明的
  • source-out：仅显示新图像与老图像没有重叠的部分，其余部分全部透明
  • source-atop：新图像仅显示与老图像重叠区域。老图像仍然可以显示
  • destination-over：新图像会在老图像的下面
  • destination-in：仅仅新老图像重叠部分的老图像被显示，其他区域全部透明
  • destination-out：仅仅老图像与新图像没有重叠的部分。 注意显示的是老图像的部分区域
  • destination-atop：老图像仅仅显示重叠部分，新图像会显示在老图像的下面
  • lighter：新老图像都显示，但是重叠区域的颜色做加处理（十六进制的每个颜色位进行加法）
  • darken：保留重叠部分最黑的像素。（十六进制的每个颜色位进行比较，得到最小的）
  • lighten：保留重叠部分最量的像素。（十六进制的每个颜色位进行比较，得到最大的）
  • xor：重叠部分会变成透明
  • copy：只有新图像会被保留，其余的全部被清除

看起来很难记是吗？其实很简单，只要记住下面几条：

  1. source 表示层级：新图形 > 老图形，destination 表示层级：新图形 < 新图形
  2. over 表示新旧图形正常覆盖，in 表示只显示重叠部分，out 表示只显示不重叠部分，atop表示只显示重叠部分和层级在下面的图形

裁剪路径（遮罩）

**clip()**，把创建好的路径变成裁剪路径，只会显示路径闭合生成的区域，外面的区域会被隐藏

只会隐藏调用 clip() 之后绘制的图片

动画

到这里，我们基本的绘图操作就都学会了。在开头说到可以搭配循环计时器等东西来实现动画效果。本质上就是不断地重绘图而已。

需要用到的循环工具就是这个函数：requestAnimationFrame()

使用时，在绘图函数里调用该函数，并传入绘图函数，实现自我调用，达成循环更新图像，即：

当然你也可以用常规的定时器 setInterval() 和 setTimeout()，不过由于他们的执行和屏幕的刷新并不一致，很容易就引起丢帧。而 requestAnimationFrame() 是由系统来决定回调函数的执行时机，它能保证回调函数在屏幕每一次的刷新间隔中只被执行一次，从而避免了丢帧现象

在写绘图代码时，要有一定的章法，基本遵循以下步骤：

  1. 清空 canvas：绘制每一帧动画之前，需要清空画布。清空最简单的做法就是 clearRect() 方法。
  2. 保存 canvas 状态：如果在绘制的过程中会更改 canvas 的状态（颜色、移动了坐标原点等），又在绘制每一帧时都是原始状态的话，则最好保存下 canvas 的状态
  3. 绘制动画图形：这一步才是真正的绘制动画帧
  4. 恢复 canvas 状态：如果你前面保存了 canvas 状态，则应该在绘制完成一帧之后恢复 canvas 状态
